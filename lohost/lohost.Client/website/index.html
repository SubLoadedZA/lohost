Welcome to your lohost site!

Guess what? This website you are looking at right now. It's running from your computer.

If you open the "App" directory, living in the folder of the lohost.exe you ran, you will see the index.html for this page. You can change the HTML in there, refresh this page and see the website immediately get updated.

You can send the URL of this site to a friend and they will be able to browse it, just like any other website.

Understanding the folder structure

If you open up the lohost folder, you should see the following folder structure:

- App
- Logs
- config.json
- lohost.exe

If you want to use lohost further, you will need to understand what each thing does.

App

This is the folder that contains all the content for your website. When you want to put your website live, put it in this folder. Make sure to include an index.html in the root (like most websites) in order to give your website a starting location.


Logs

This folder contains all the logs from the running of your website. The logs will give you insight into what is being called as well as valuable debugging information.


config.json

This file is created when you run lohost.exe for the first time. It contains all the configuration options available for your site, including the name of your website subdomain.

You can see the config options via the "View config options" link. In order to have the changes reflected, you will need to stop and start the lohost.exe.


lohost.exe

This is the main executable for lohost. This application needs to be running in order to make your website available.

Moving/distributing your website

The lohost application is entirely self-contained to the lohost folder. If you want to move the application to another computer/folder, you just need to move the lohost folder.

If you copy the lohost folder to another computer/folder and then run the lohost.exe it will become a backup for your website. If you stop one lohost.exe the other will take over and this can be done across any number of computers.

You can also make different routes be served by different lohost.exe instances. If you want https://{yourURL}.lohost.io/route1 to be served from computer and https://{yourURL}.lohost.io/route2 be served from another, this can be done via the config.json file.

Limitations

Currently, the only file types supported are those that are used to run a website, so .css, .html, .js, .jpg, .png, .gif and .json. This is pretty easy to extend in the future to any type of file type but those are a good starting point.

Also, documents are restricted to 1MB. This is configured in the lohost.io website API and can potentially be made infinitely large. Sharing is caring though and the limit is placed as to make lohost.io open to all. Should you want to run your own lohost API, you can open it to any amount!
